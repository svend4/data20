## 🎯 Шаг за шагом с описанием каждого элемента

---

## Шаг 1: Открыть главную страницу репозитория

### URL:
```
https://github.com/svend4/data20
```

### Что вы увидите:
На главной странице репозитория вверху есть несколько вкладок:
- **Code** (Код) - текущая вкладка
- **Issues** (Задачи)
- **Pull requests** (Запросы на слияние)
- **Actions** (Автоматизация)
- **Projects** (Проекты)
- И другие...

## Шаг 2: Найти раздел "Releases" в боковой панели

### Где искать:


### Что нажимать:
1. Найдите секцию **"Releases"** с иконкой 🏷️
2. Нажмите на **цифру** (например, "1") или на слово **"release"**
3. Или нажмите на саму надпись **"Releases"**

## Шаг 3: Страница Releases

После нажатия вы попадете на страницу:
```
https://github.com/svend4/data20/releases

```

```
│  Releases                                            │
│                                                      │

│  │  [Draft a new release]  (зеленая)   │ ← КНОПКА! │

│                                                      │
│  ┌──────────────────────────────────────┐          │
│  │  📱 Data20 Mobile v1.0.0             │          │
```

### Кнопки на этой странице:

1. **"Draft a new release"** (справа вверху, ЗЕЛЕНАЯ)
   - Для создания НОВОГО релиза
   - Прямая ссылка: https://github.com/svend4/data20/releases/new

2. **"View release"** (под существующим релизом)
   - Посмотреть подробности релиза v1.0.0
   - URL: https://github.com/svend4/data20/releases/tag/v1.0.0

3. **"Edit"** (рядом с "View release")
   - Редактировать существующий релиз
   - Можно изменить описание или добавить файлы

## Шаг 4: Создать новый Release

### Нажмите "Draft a new release"

Вы попадете на форму создания релиза:

```
┌──────────────────────────────────────────────────────┐
│  Create a new release                                │
├──────────────────────────────────────────────────────┤
│                                                      │
│  Choose a tag                                       │
│  ┌──────────────────────────────────────┐          │
│  │ v1.0.1                    [▼]        │ ← Версия │
│  └──────────────────────────────────────┘          │
│  [Create new tag: v1.0.1 on publish]              │
│                                                      │
│  Target: main                                       │
│                                                      │
│  Release title                                      │
│  ┌──────────────────────────────────────┐          │
│  │ Data20 Mobile v1.0.1                 │ ← Название│
│  └──────────────────────────────────────┘          │
│                                                      │
│  Describe this release                              │
│  ┌──────────────────────────────────────┐          │
│  │                                      │          │
│  │  Скопируйте текст из                │ ← Описание│
│  │  RELEASE_NOTES.md                    │          │
│  │                                      │          │
│  └──────────────────────────────────────┘          │
│                                                      │
│  Attach binaries by dropping them here              │
│  ┌──────────────────────────────────────┐          │
│  │  📎 Перетащите APK файл сюда        │ ← Файлы  │
│  └──────────────────────────────────────┘          │
│                                                      │
│  ☑️ Set as the latest release                       │
│  ☐ Set as a pre-release                            │
│  ☐ Create a discussion for this release            │
│                                                      │
│  [Publish release] (ЗЕЛЕНАЯ КНОПКА)                │
│                                                      │
└──────────────────────────────────────────────────────┘
```

---

## Шаг 5: Заполнить форму

### 1. Choose a tag (Выбор тега):
```
Введите: v1.0.1
```
Система покажет: "Create new tag: v1.0.1 on publish"

### 2. Release title (Название):
```
Введите: Data20 Mobile v1.0.1
```
Или любое другое название

### 3. Describe this release (Описание):
```
Скопируйте содержимое из файла RELEASE_NOTES.md
```
Или напишите свое описание изменений

### 4. Attach binaries (Прикрепить файлы):
Если у вас есть APK файл:
- Перетащите его в область "Attach binaries"
- Или нажмите на область и выберите файл

### 5. Опции внизу:
- ✅ **Set as the latest release** - ОСТАВИТЬ ВКЛЮЧЕННЫМ
- ⬜ **Set as a pre-release** - НЕ ВКЛЮЧАТЬ
- ⬜ **Create a discussion** - по желанию

---

## Шаг 6: Опубликовать

### Нажмите "Publish release" (зеленая кнопка внизу)

После публикации:
1. Релиз появится на https://github.com/svend4/data20/releases
2. GitHub Actions запустится автоматически (если настроен)
3. Пользователи смогут скачать релиз

## 🔍 Альтернативные способы попасть на страницу Releases

### Способ 1: Через URL
Просто откройте:
```
https://github.com/svend4/data20/releases
```

### Способ 2: Через вкладку Code
1. Нажмите на вкладку **"Code"** (если вы не на ней)
2. Справа найдите **"Releases"**
3. Нажмите на цифру или "release"

### Способ 3: Через верхнее меню
На некоторых репозиториях есть вкладка **"Releases"** в верхнем меню рядом с Code, Issues, Pull requests

### Способ 4: Напрямую создать release
Сразу перейти к созданию:
```
https://github.com/svend4/data20/releases/new
```

## 🎓 Полезная информация

### Что такое Release?
Release (релиз) - это версия вашего проекта, которую можно скачать.

### Зачем нужны Releases?
- Пользователи могут скачать конкретную версию
- Можно прикрепить готовые файлы (APK, EXE, ZIP)
- Удобно отслеживать историю версий
- Автоматическое оповещение подписчиков

### Кто может создавать Releases?
Создавать releases могут:
- Владелец репозитория
- Администраторы
- Участники с правами записи (write access)
